# 代码质量优化

## 评估结论

整体架构清晰，分层合理，以下是发现的具体问题：

---

## 优化项（按优先级排列）

### P0 - 响应性 Bug

`**CreateOutputModal.vue`：`isValid` 是普通函数，在模板中被多次调用**

模板里 `:disabled="!isValid()"` 在每次渲染都重新执行函数调用。应改为 `computed`，利用 Vue 的缓存机制，且语义更清晰：

```js
// 改前
const isValid = () => platform.value.trim() && url.value.trim()

// 改后（需同时引入 computed）
const isValid = computed(() => !!(platform.value.trim() && url.value.trim()))
```

---

### P1 - 性能问题

`**NoteEditorModal.vue`：`MarkdownIt` 实例在 `setup()` 内部创建**

每次弹窗打开都会执行 `new MarkdownIt(...)` 创建新实例，浪费初始化开销。应提升到模块顶层作为单例：

```js
// 文件顶部（import 之后，setup 之外）
import MarkdownIt from 'markdown-it'
const md = new MarkdownIt({ breaks: true, linkify: true })

// setup 内部移除该行
```

---

### P2 - 用户体验

`**TodoView.vue`：页面初始加载缺少 loading 状态**

`onMounted` 异步拉取任务期间 `tasks` 为空数组，用户会看到"没有符合条件的任务"的空状态后内容突然填充，体验不连贯。

新增 `loading` ref，在模板中加一个加载占位：

```js
const loading = ref(false)
onMounted(async () => {
  loading.value = true
  try { ... } finally { loading.value = false }
})
```

模板中任务列表区：

```html
<!-- loading 骨架 -->
<div v-if="loading" class="flex flex-col gap-3">
  <div v-for="i in 3" :key="i" class="h-14 rounded-xl bg-white/5 animate-pulse" />
</div>
<!-- 原有的 template v-if / v-else -->
<template v-else-if="filteredTasks.length > 0"> ... </template>
<div v-else> ... 空状态 ... </div>
```

---

### P3 - 代码组织

`**TodoView.vue`：`handleCreateNote` 在声明前引用了后方的变量**

`handleCreateNote` 函数内部使用了 `noteForModal`、`noteTaskState`、`isNoteModalOpen`，但这三个 `ref` 在文件中定义于该函数之后（第 88-91 行）。虽然 `<script setup>` 的单一作用域不会产生运行时错误，但违反了"先声明再使用"的可读性原则。

将 `handleCreateNote` 函数体移到笔记弹窗相关 ref 声明之后（即第 88 行之后），或将笔记相关 ref 上移到 `handleCreateNote` 之前。

---

### P4 - API 层统一错误处理

`**request.js`：缺少响应拦截器**

当前每个 `try/catch` 独立打印错误，无法统一追踪。在 `[src/api/request.js](src/api/request.js)` 中添加响应拦截器：

```js
request.interceptors.response.use(
  res => res,
  err => {
    console.error('[API Error]', err.config?.url, err.response?.status, err.message)
    return Promise.reject(err)
  }
)
```

---

### P5 - 模板冗余

`**TaskItem.vue`：NOTED 和 DONE 状态的"查看笔记"按钮代码完全重复**

两段完全一样的 Tailwind class 字符串出现在 NOTED 和 DONE 两个分支，提取为 script 中的常量：

